# 最简单练手项目：小红书标题生成器

## 1) 这个项目要做什么
做一个**单页网页工具**，用户输入一个主题（例如“减脂早餐”），点击按钮后，网页会自动生成几条风格不同的小红书标题候选。

目标是先学会前端最基础三件套：
- HTML：页面结构
- CSS：页面样式
- JavaScript：按钮点击和标题生成逻辑

这个版本不接后端、不接数据库、不接账号系统，保证你可以在本地直接打开就能用。

---

## 2) 需要哪些文件
建议只用 3 个文件：

1. `index.html`
2. `style.css`
3. `script.js`

（可选）第 4 个文件：
4. `README.md`

---

## 3) 每个文件有什么作用

### `index.html`
- 定义页面骨架：标题、输入框、按钮、结果区域。
- 把 `style.css` 和 `script.js` 连接起来。

### `style.css`
- 控制视觉效果：字体、颜色、按钮样式、间距。
- 让页面在 iPhone 上也能正常显示（基础响应式）。

### `script.js`
- 读取输入内容。
- 用简单模板拼接出标题（例如“主题 + 数字 + 情绪词”）。
- 把生成结果渲染到页面。

### `README.md`（可选但推荐）
- 写清楚项目介绍、如何运行、后续可扩展点。
- 未来你发给别人时，对方一看就知道怎么用。

---

## 4) 一步一步创建它（Windows + iPhone 场景）

> 你现在只需要会“新建文件 + 复制粘贴 + 在浏览器打开”。

### 第 0 步：准备工具
- 安装 VS Code（Windows）。
- 安装一个浏览器（Chrome/Edge 都行）。
- 准备一个项目文件夹，例如：`D:\codex-practice\xiaohongshu-title-tool`。

### 第 1 步：创建项目文件夹
1. 在 Windows 资源管理器中新建文件夹：`xiaohongshu-title-tool`。
2. 用 VS Code 打开这个文件夹。

### 第 2 步：新建 3 个文件
在 VS Code 左侧文件区，创建：
- `index.html`
- `style.css`
- `script.js`

（可选）再建 `README.md`。

### 第 3 步：先只搭页面，不写复杂逻辑
在 `index.html` 里先放 4 个区域：
1. 页面主标题（例如：小红书标题生成器）
2. 一个输入框（输入主题）
3. 一个按钮（生成标题）
4. 一个结果列表区域（显示生成结果）

这一步只求“看得见页面”。

### 第 4 步：加基础样式
在 `style.css` 里做最少样式：
- 页面居中
- 输入框和按钮有间距
- 结果区域一条一条显示

这一步只求“比默认样式好看一点”。

### 第 5 步：加最简单生成逻辑
在 `script.js` 里做 3 件事：
1. 按钮点击时，读取输入框内容。
2. 用 3~5 个固定模板生成标题。
3. 把标题显示到结果区域。

建议模板先写死，例如：
- `我靠这个【主题】真的有用！`
- `新手必看：3步搞定【主题】`
- `别再乱试了，这样做【主题】更高效`

### 第 6 步：本地运行
- 直接双击 `index.html`，浏览器会打开网页。
- 输入主题，点击按钮，看是否出现标题。

### 第 7 步：用 iPhone 测试看显示
最简单方式：
- 把项目上传到 GitHub Pages / Vercel（后续再做）。
- 用 iPhone Safari 打开链接，检查按钮和文字是否挤在一起。

你暂时也可以先不做这一步，先把 PC 版跑通。

### 第 8 步：练手升级（按顺序）
当你跑通后，再逐个加：
1. “复制标题”按钮
2. “再来一组”按钮
3. 标题风格切换（干货/情绪/避坑）
4. 输入为空时提示文案

---

## 给新手的成功标准（今天就能达成）
只要满足下面 3 条，你就已经成功完成第一个网页工具：
1. 页面能打开；
2. 输入主题后点按钮能出现 3 条标题；
3. 刷新后还能继续使用。

这就是一个非常好的 Codex 练手项目起点。
